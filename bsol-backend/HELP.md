## 1. What is your project about?

My project, Bachelor Solution, is a platform for bachelors to find rooms, mess services, food stalls, roommates, and open posts in one place. The main idea is to reduce the problem of searching across multiple Telegram groups or scattered websites.

---

## 2. What problem does your project solve?

Many bachelors struggle to find accommodation and daily-life services quickly. Most of the time, they depend on Telegram groups, which can be messy and unreliable. My project solves this by bringing all essential listings into one organized platform with search and filters.

---

## 3. Why did you choose this project?

I chose this project because it solves a real-life problem that many students and bachelors face daily. It also gave me a chance to build a complete full-stack application using both frontend and backend technologies.

---

## 4. What technologies did you use in this project?

I used Next.js for the frontend because it helps create a modern and responsive user interface. For the backend, I used Spring Boot to create REST APIs and handle business logic. I also used a database to store listings and related data.

---

## 5. What is the main feature of your project?

The main feature is centralized listing management. Users can browse listings, search by location or category, and post their own listings. It makes the whole process faster and more organized.

---

## 6. What makes your project different from other listing platforms?

This project is specifically focused on bachelors’ daily needs such as rooms, mess, food stalls, roommates, and open posts. It is more targeted and practical for that user group, rather than being a general marketplace.

---

## 7. What challenges did you face while building this project?

One of the main challenges was managing different listing types using one unified structure. I also had to make the backend flexible enough to support search, filters, and image uploads for multiple categories.

---

## 8. How did you handle the backend logic?

I created REST APIs in Spring Boot to handle listing creation, retrieval, updating, and deletion. The backend is also designed to support filtering and category-based data retrieval efficiently.

---

## 9. What did you learn from this project?

I learned how to build a complete full-stack project, connect frontend and backend, work with APIs, manage data, and solve real-world user problems through software development.

---

## 10. How would you improve this project in the future?

I would improve it by adding user authentication, chat or contact features, rating/review systems, and better recommendations based on location and preferences.
